Product Backend gRPC Project
This project implements a gRPC-based backend for managing products and user authentication using Go, PostgreSQL, and GORM. It includes services for product CRUD operations and user login with JWT authentication.
Prerequisites
Before running the project, ensure you have the following installed:

Go (version 1.18 or higher): Download
PostgreSQL (version 12 or higher): Download
protoc (Protocol Buffers compiler): Installation Guide
Postman (version 9.0 or higher, for testing gRPC): Download
grpcurl (optional, for testing gRPC via CLI): Installation Guide

Project Structure
product-backend/
├── config/                           # Configuration loading (e.g., database settings)
├── database/                      # Database init
├── grpc/                              # gRPC service definitions and generated code
│   ├── product.proto            # Proto file defining services
│   ├── product.pb.go           # Generated by protoc
│   ├── product_grpc.pb.go  # Generated by protoc
├── handlers/                        # gRPC handlers for ProductService and AuthService
├── models/                          # GORM models (User, Product)
├── repository/                      # Database operations
├── services/                         # Business logic for ProductService and AuthService
├── utils/                                # Utility functions (e.g., JWT generation)
├── go.mod                            # Go module dependencies
├── main.go                           # Main application entry point

Setup Instructions

1. Clone the Repository
   git clone `<repository-url>`
   cd product-backend
2. Initialize Go Module
   Ensure the go.mod file exists with the following content:
   module product-backend

go 1.18

require (
    github.com/dgrijalva/jwt-go v3.2.0+incompatible
    golang.org/x/crypto v0.0.0-20221012134737-56aed0617322
    google.golang.org/grpc v1.50.0
    gorm.io/driver/postgres v1.4.0
    gorm.io/gorm v1.24.0
)

Run the following command to sync dependencies:

go clean -modcache
go mod tidy
go mod vendor

3. Configure PostgreSQL

Install and start PostgreSQL on your machine.

Create a database (e.g., mydb):
psql -U postgres
CREATE DATABASE mydb;

Create tables for users and products:Save the following SQL to base_schema.sql:
-- Create table users
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(255) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP WITH TIME ZONE
);

-- Create table products
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    price DOUBLE PRECISION NOT NULL,
    quantity INTEGER NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP WITH TIME ZONE
);

Run the SQL:
psql -U postgres -d mydb -f base_schema.sql

Create a config/config.go file to load database configuration:
package config

import "os"

type Config struct {
    DBHost     string
    DBUser     string
    DBPassword string
    DBName     string
    DBPort     string
}

func LoadConfig() (*Config, error) {
    return &Config{
        DBHost:     os.Getenv("DB_HOST"),
        DBUser:     os.Getenv("DB_USER"),
        DBPassword: os.Getenv("DB_PASSWORD"),
        DBName:     os.Getenv("DB_NAME"),
        DBPort:     os.Getenv("DB_PORT"),
    }, nil
}

Set environment variables for database connection:
set DB_HOST=localhost
set DB_USER=postgres
set DB_PASSWORD=your_password
set DB_NAME=mydb
set DB_PORT=5432

4. Generate gRPC Code

Ensure product.proto exists in the grpc/ directory:
syntax = "proto3";

package grpc;
option go_package = "./";

service ProductService {
  rpc CreateProduct (CreateProductRequest) returns (ProductResponse);
  rpc GetProduct (GetProductRequest) returns (ProductResponse);
  rpc UpdateProduct (UpdateProductRequest) returns (ProductResponse);
  rpc DeleteProduct (DeleteProductRequest) returns (DeleteProductResponse);
  rpc ListProducts (ListProductsRequest) returns (ListProductsResponse);
}

service AuthService {
  rpc Login (LoginRequest) returns (LoginResponse);
}

message CreateProductRequest {
  string name = 1;
  float price = 2;
  int32 quantity = 3;
}

message UpdateProductRequest {
  int32 id = 1;
  string name = 2;
  float price = 3;
  int32 quantity = 4;
}

message GetProductRequest {
  int32 id = 1;
}

message DeleteProductRequest {
  int32 id = 1;
}

message ListProductsRequest {}

message ProductResponse {
  int32 id = 1;
  string name = 2;
  float price = 3;
  int32 quantity = 4;
}

message DeleteProductResponse {
  bool success = 1;
}

message ListProductsResponse {
  repeated ProductResponse products = 1;
}

message LoginRequest {
  string username = 1;
  string password = 2;
}

message LoginResponse {
  string token = 1;
}

Generate gRPC code:
cd grpc
protoc --go_out=. --go-grpc_out=. product.proto

5. Run the Application

Start the gRPC server:
cd .\product-backend
go run main.go

The server will start on localhost:50051 and create sample data (users admin/admin123, user1/user123, and products Laptop, Smartphone, Headphones).

Verify the server is running:

Look for the log: gRPC server is running on port :50051.

6. Test gRPC Endpoints
   Using Postman

Open Postman (version 9.0 or higher).
Create a new gRPC request:
Click New > gRPC Request.
Enter Server URL: localhost:50051.
Click Select a method > Import a .proto file.
Choose .\product-backend\grpc\product.proto.

Select the grpc.AuthService/Login method.
Enter the request body:{
    "username": "admin",
    "password": "admin123"
}

Click Invoke to send the request.
Check the response (should return a JWT token):{
    "token": "your_jwt_token_here"
}

Using grpcurl (Optional)

Test the Login endpoint:grpcurl -plaintext -proto .\product-backend\grpc\product.proto -d '{"username":"admin","password":"admin123"}' localhost:50051 grpc.AuthService/Login

Test the ListProducts endpoint:grpcurl -plaintext -proto .\product-backend\grpc\product.proto localhost:50051 grpc.ProductService/ListProducts

Troubleshooting

Database Connection Errors:

Ensure PostgreSQL is running and environment variables (DB_HOST, DB_USER, DB_PASSWORD, DB_NAME, DB_PORT) are set correctly.
Check tables:psql -U postgres -d mydb -c "\dt"
psql -U postgres -d mydb -c "SELECT * FROM users;"
psql -U postgres -d mydb -c "SELECT * FROM products;"

gRPC Errors:

If Postman reports "Method not found" or "Failed to connect", verify the server is running and the .proto file is imported correctly.
Ensure product_grpc.pb.go contains package grpc and the Register* functions.

Duplicate Data:

The CreateDefaultUser and CreateDefaultProduct functions in main.go check for existing entries to avoid UNIQUE constraint failed errors.

gRPC Reflection (Optional):

If you prefer using grpcurl without -proto, enable reflection in main.go:import "google.golang.org/grpc/reflection"
// ...
reflection.Register(grpcServer)

Then test without -proto:grpcurl -plaintext -d '{"username":"admin","password":"admin123"}' localhost:50051 grpc.AuthService/Login

Notes

The project uses bcrypt for password hashing and jwt-go for authentication tokens.
Ensure the utils package contains a GenerateJWT function for token generation.
For production, consider disabling gRPC reflection and securing the database connection with SSL.
